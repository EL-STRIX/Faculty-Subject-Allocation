# Contributing to Faculty Subject Allocation System

Thank you for your interest in contributing to the **Faculty Subject Allocation System**! Your help makes this project better for Brainware University's Computational Sciences Department.

Please take a moment to review this document before submitting your first pull request.

## 🚀 How Can I Contribute?

### Reporting Bugs
- Use the project's **GitHub Issues** tab to report bugs.
- Provide a clear and descriptive title.
- Describe the exact steps which reproduce the problem.
- Include screenshots if it's a UI issue.

### Suggesting Enhancements
- Enhancement suggestions are tracked as GitHub issues.
- Explain why this enhancement would be useful to other users.
- Describe the feature you'd like to see.

### Pull Requests
1. **Fork** the repository.
2. Create a new **branch** (`git checkout -b feature/amazing-feature`).
3. Make your changes in the branch.
4. Ensure your code follows the project's coding standards.
5. **Commit** your changes (`git commit -m 'Add amazing feature'`).
6. **Push** to the branch (`git push origin feature/amazing-feature`).
7. Open a **Pull Request**.

## 🛠️ Development Guidelines

- **Vanilla PHP**: Keep the backend logic simple and efficient. Use PDO or MySQLi (Current project uses MySQLi).
- **CSS Styling**: Maintain the custom design system using vanilla CSS. Avoid adding heavy frameworks unless necessary.
- **Documentation**: If you add a new feature, please update the `README.md` accordingly.
- **Validation**: Ensure both frontend and backend validation are robust to prevent duplicate entries or SQL injection.

## ⚖️ Code of Conduct
Please be respectful and professional in all your interactions with the project and other contributors.
